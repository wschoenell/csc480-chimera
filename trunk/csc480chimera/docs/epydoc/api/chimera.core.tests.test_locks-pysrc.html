 53              """Rationale: We use 5 threads for each method (locked and 
 54              unlocked). As unlocked methods isn't serialized, they runs 
 55              'at the same instant', while locked methods will be 
 56              serialized and will run only when the previous one

 58              then returns the time of completion (with an arbitrary 
 59              zero point to give small numbers). The deviation from the 
 60              mean of unlocked methods termination times should be 
 61              nearly zero, as every methods runs at the same time. For 
 62              locked ones, the termination time will be a linear 
 63              function with the slope equals to the load (sleep in this 
 64              case), and as we use 10 threads for the locked case, the 
 65              deviation will be ~ 2.872. We use a simple equals_eps to 
 66              handle load factors that may influence scheduler 
 67              performance and timmings.
